# Política de Segurança

Obrigado por ajudar a manter o projeto seguro.

1) Como reportar vulnerabilidades
- Abra uma issue com a tag `security` no repositório indicando: resumo, passos para reproduzir, versão afetada e impacto esperado.
- Se a vulnerabilidade for sensível, envie uma mensagem privada para os mantenedores (use o canal/contato listado no repositório) em vez de publicar detalhes imediatamente.

2) Informações a incluir
- Versão do Gargantua Launcher
- Sistema operacional e versão
- Logs relevantes e passos para reproduzir
- Possível correção ou mitigação sugerida (se conhecida)

3) Não inclua
- Chaves privadas, senhas ou credenciais reais nos relatórios públicos. Use dados fictícios ao explicar um fluxo.

4) Política de divulgação
- Recebemos, analisamos e aplicamos correções com prioridade conforme a gravidade.
- Quando aplicável, faremos divulgação pública após correção ou acordo com o reporter.

5) Versões suportadas
- Versão corrente (código no branch `main`) é a principal suportada. Para outras versões, a resposta pode variar.

Obrigado por contribuir para um projeto mais seguro.
